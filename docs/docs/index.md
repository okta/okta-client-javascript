## Design Philosophy

At Okta, we strive for architectural parity across all of our client SDKs, regardless of language or environment. See the [Okta SDK Design](https://okta-client-design.netlify.app/design/) site for more details.

Okta Client JavaScript is developed with a developer-first approach. Our goal is to enable developers to seamlessly incorporate OAuth2 into new and (perhaps more importantly) existing applications without requiring major refactoring. We carefully consider how a developer would use any new class (pun fully intended), API, or paradigm before committing to the approach and merging it into the ecosystem.
